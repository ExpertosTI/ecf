Integración ECF

Guía técnica para la emisión y recepción de Comprobantes Fiscales Electrónicos. Nuestra API REST permite automatizar todo el flujo con DGII.

1. Autenticación

Todas las peticiones deben incluir el header de autorización con tu API Key.

Authorization: Bearer sk_live_your_api_key

2. Emisión de e-CF

POST /api/v1/emitir

Envía la estructura del e-CF para firma y envío a DGII.

Request Body (JSON)
{ "rnc_emisor": "133109192", "tipo_ecf": "31", "ncf": "E310000000001", "monto_total": 1500.00, "items": [ { "desc": "Producto A", "precio": 1271.19, "itbis": 228.81 } ] }

Probador en Vivo

Envía un payload de prueba al ambiente de Sandbox.

Verificación de Firma

Nuestra API envía un hash HMAC-SHA256 para validar que la petición viene de Renace.

const crypto = require('crypto'); const secret = 'your_webhook_secret'; const signature = req.headers['x-renace-signature']; const expected = crypto.createHmac('sha256', secret) .update(JSON.stringify(req.body)) .digest('hex');
