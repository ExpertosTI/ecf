# Instrucciones Personalizadas para el Proyecto ECF

## 🚨 Reglas de Oro Operacionales (Antigravity Rules)

### 1. Conexiones Externas y SSH
- **PROHIBIDO ejecutar comandos de red/SSH**: Bajo ninguna circunstancia ejecutes comandos como `ssh`, `scp`, `rsync` o similares a través de herramientas de terminal del agente.
- **Delegación de comandos al Operador**: Todo comando destinado a los servidores remotos debe ser escrito de forma explícita y real para que el usuario lo ejecute manualmente en sus terminales SSH correspondientes.

### 2. Consulta de Documentación
- **Consulta Obligatoria**: Antes de realizar cualquier sugerencia, cambio en el código o proponer comandos, consulta obligatoriamente `AVANCES_Y_PENDIENTES.md` y `README.md`.
- **Actualización de la Documentación**: Cuando se complete una tarea, migración o cambio relevante, actualiza inmediatamente la documentación para que refleje el nuevo estado real de la plataforma.

### 3. Rigor Quirúrgico en Comandos
- **No asumir**: Está prohibido adivinar puertos, nombres de servicios, bases de datos o rutas de carpetas.
- **Comandos Reales**: Nunca ofrezcas comandos con variables genéricas o placeholders de ejemplo (como `<schema>`, `<container_id>`, etc.). Usa siempre datos contextuales reales recuperados durante la sesión.
- **Paso a paso**: Todo el desarrollo y despliegue debe seguir un enfoque estrictamente secuencial. No te anticipes a pasos futuros hasta que el paso actual esté verificado.

### 4. Edición de Módulos Odoo y Prevención de RPC_ERROR
- **Edición Local**: El código de los módulos de Odoo debe editarse en local. Indica al operador que suba los archivos editados a través de FileZilla (u otro gestor FTP). No intentes realizar modificaciones de archivos del módulo directamente en los servidores vía comandos SSH.
- **Prevención de ParseError**: Ten especial cuidado con la compatibilidad de selectores XPath y vistas heredadas entre versiones de Odoo (por ejemplo, `h2` en Odoo 18 vs `h3` en Odoo 19). Asegura compatibilidad estricta para evitar interrupciones de base de datos (`RPC_ERROR`).
